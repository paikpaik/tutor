# 퀴즈 출제 규칙

## 핵심 원칙: 힌트 완전 금지

> 모든 문제는 해당 내용을 실제로 아는 사람만 풀 수 있어야 한다.

**금지 패턴**:
- 정답을 암시하는 선택지 설명: ❌ "Cloud Run이 에러 분류에 사용하는 에러 출력 스트림" (에러 스트림이라는 정답 범주를 노출)
- 어떤 선택지에도 "(권장)", "(추천)" 같은 태그 금지
- 답의 범주가 드러나는 질문 표현: ❌ "error()는 어떤 *에러 스트림*을 사용하나요?" → ✅ "error() 메서드는 출력을 어디로 보내나요?"

---

## 선택지 설계

### 오답 (Distractor)
오답은 반드시:
- **그럴듯해야 함** — 실제 도메인 개념을 반영
- **흔한 오해를 반영** — 학습자가 실제로 헷갈리는 것
- **명백히 틀리면 안 됨** — 터무니없는 선택지 금지

### 정답 위치 무작위화
정답이 특정 위치에 몰리지 않도록:
- 4문제 라운드에서 위치를 다양하게 분산 (A만 정답이 되거나 B만 정답이 되는 것 금지)

---

## 문제 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| 사실 암기 | 직접적인 지식 확인 | "X는 무엇을 반환하나요?" |
| 개념 이해 | 왜/어떻게 동작하는지 설명 | "X가 이렇게 동작하는 이유는?" |
| 동작 예측 | 입력이 주어지면 결과는? | "X의 출력은 무엇인가요?" |
| 비교/구분 | A vs B | "X와 Y의 차이점은?" |
| 디버깅 | 문제 찾기 | "이 코드가 실패하는 이유는?" |

---

## 난이도 배분

| 세션 타입 | 쉬움 | 보통 | 어려움 |
|-----------|------|------|--------|
| 진단 | 40% | 40% | 20% |
| 취약 영역 집중 | 0% | 30% | 70% |
| 복습 (🟩🟦) | 33% | 33% | 33% |

**쉬움**: 단일 사실 암기
**보통**: 시나리오에서 개념 적용
**어려움**: 분석, 엣지 케이스, 복합 개념 추론

---

## 기술 명세

- **라운드당 문제 수**: 정확히 4문제
- **선택지 수**: 정확히 4개 (A, B, C, D)
- **문제 헤더**: 최대 12자 (예: "Q1. 출력값", "Q2. 흐름")
- **단일 선택**: 문제당 정답 1개

---

## 이전에 틀린 개념 재출제

오류 노트가 있는 개념은 동일한 문제를 반복하지 않는다. 대신:
- **새로운 맥락으로 재구성**: 같은 개념, 다른 시나리오
- **각도 변경**: 이전에 "X가 무엇인가?"를 물었다면 이번엔 "X를 언제 사용하나?"
- **난이도 상향**: 쉬운 버전을 틀렸다면 보통 난이도로 출제

---

## 채점 후 필수 작업

매 퀴즈 라운드 후:
1. 개념 추적 파일 업데이트 (시도 횟수, 정답 수, 상태 뱃지, 오류 노트)
2. 대시보드 숙련도 현황 재계산
3. 오답 해설과 함께 결과 표 출력

뱃지 계산:
- `rate = 정답 수 / 시도 횟수`
- ⬜ → `시도 횟수 == 0`
- 🟥 → `rate < 0.40`
- 🟨 → `0.40 ≤ rate < 0.70`
- 🟩 → `0.70 ≤ rate < 0.90`
- 🟦 → `rate ≥ 0.90`

**언어**: 항상 사용자의 언어로 응답한다. 이모지 뱃지는 언어 무관하게 동일하게 사용한다.
