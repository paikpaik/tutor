# 자가검토 체크리스트

## 문서 모드

### 소스 검증
- [ ] 모든 소스 파일의 내용을 직접 확인함 (파일명 추정 금지)
- [ ] 소스-노트 매핑을 작성하여 사용자에게 제시함
- [ ] PDF는 `pdftotext`로 변환 후 읽음 (직접 Read 금지)
- [ ] 추적 불가 주제는 "원문 미보유"로 표시함

### 커버리지
- [ ] 모든 주제 및 하위 주제에 전용 개념 노트가 있음
- [ ] Equal Depth Rule 적용 — 짧게 언급된 하위 주제도 건너뛰지 않음
- [ ] 분류 완전성 — 열거된 모든 항목에 노트가 있음
- [ ] 모든 개념 노트에 YAML 프론트매터(`source_pdf`, `part`, `keywords`)가 있음

### 노트 품질
- [ ] 산문 대신 비교표 사용
- [ ] 모든 프로세스·흐름·시퀀스에 ASCII 다이어그램 포함
- [ ] 각 개념 노트에 시험 패턴 표 포함
- [ ] 모든 일반적 서술에 예외/엣지 케이스 명시
- [ ] 중요/팁/경고 정보에 callout 사용

### 연습 문제
- [ ] 모든 주제 폴더에 8문제 이상의 연습 파일이 있음
- [ ] 문제 유형 분포: 암기 ≥60%, 적용 ≥20%, 분석 ≥2개
- [ ] 모든 정답은 `> [!answer]- 정답 보기` fold callout으로 숨겨져 있음
- [ ] 질문 선택지나 문구에 힌트 없음
- [ ] `## 관련 개념` 섹션에 wiki-link 포함

### 상호 링크
- [ ] 모든 개념 노트에 `## 관련 노트` 있음
- [ ] MOC가 모든 개념 노트 및 연습 파일에 링크
- [ ] 개념 노트 ↔ 연습 문제 상호 링크
- [ ] Quick Reference 헤딩이 개념 노트에 링크
- [ ] Exam Traps가 개념 노트에 링크

### 대시보드
- [ ] MOC가 wiki-link와 함께 모든 주제를 포함
- [ ] Quick Reference에 모든 핵심 공식과 비교표 포함
- [ ] Exam Traps가 주제별 주요 함정 포인트를 다룸
- [ ] Tag Index에 등록된 모든 태그 포함

## 코드베이스 모드

### 프로젝트 커버리지
- [ ] 모든 주요 모듈에 전용 노트가 있음
- [ ] 전체 시스템 다이어그램이 포함된 아키텍처 개요 작성
- [ ] 엔드투엔드 요청 흐름 문서화
- [ ] 기술 스택 식별 및 목록화

### 모듈 노트
- [ ] 모든 모듈 노트에 YAML 프론트매터(`module`, `path`, `keywords`) 있음
- [ ] Purpose, Key Files, Public Interface, Internal Flow, Dependencies 모두 포함
- [ ] 의존 모듈에 대한 상호 참조 있음

### 연습 문제
- [ ] 주요 모듈별 최소 5개의 연습 문제
- [ ] 코드 읽기, 설정, 디버깅, 확장 유형 포함
- [ ] 모든 정답은 fold callout으로 숨겨져 있음

### 대시보드
- [ ] Getting Started 섹션이 실행 가능한 명령어로 구성됨 (복붙 가능)
- [ ] Module Map이 모든 모듈을 포함
- [ ] 권장 온보딩 경로가 명확하게 정의됨

### 상호 링크
- [ ] 아키텍처 노트가 구현 모듈을 참조
- [ ] 연습 문제가 해당 모듈 노트를 참조
- [ ] 모든 모듈이 양방향 상호 참조됨

### 공통
- [ ] 태그: 영문, 소문자, kebab-case만 사용
- [ ] 모든 파일이 CWD 내에 있음
